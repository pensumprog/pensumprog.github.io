Programmering-B-valgfag-august-2017.pdf

Her følger en oversigt over pensum i programmering B, for klasserne 3i og 2i på HCØ Lyngby. Oversigten er baseret på anvendelse af Processing i undervisningen. Oversigten er vejledende og der forekommer mangler:

  • Stepwise Improvement:
    1. lav lille ændring
    2. test ændring
    3. ret koden hvis den ikke virker
    4. gentag
  • Udvalgte clean code regler:
    • god navngivning
    • ingen gentagelser
    • gør koden simpel
    • ryd altid op i koden efter en ændring
  • Objektorienteret analyse og design (forkortet OOAD): De fem trin
    1. Krav: Krav spec. og design. spec.
    2. Analyse: Beskrivelse af hvordan systemet virker uden at tænke på kode. Lav ting-forklarings liste, Brugerscenarie, Aktivitets(eller flowchart)
    3. Design: Design af hvordan det skal programmeres. Lav f.eks. klasseliste, kollaborations diagram, rollediagram og tilsidst klasse-diagrammet
    4. Programmering: Den endelige kodning (brug f.eks. clean code og stepwise improvement)
    5. Test: F.eks. system test der viser at krav og design er opfyldt
  • print til konsol
  • anvend debugger
  • skriv unit test (ikke pensum)

Det er en ufuldstændig liste, men det mest vigtige

  • Hvad er keywords i Java: Reserverede ord som sproget bruger, og ikke kan bruges som f.eks. variabel-navne, f.eks. "if"
  • Java vs. andre sprog: Typestærkt. Kompileres til bytecode, der eksekveres i tn platformsuafhængigt-program kaldet java-virtual-machine
  • Exceptions og behandling af exceptions: Undtagelser som sker runtime, fangestry/catch
  • Processing Metoder/Funktioner: setup, draw, mousePressed, keyPressed, size, background, clear, frameRate, random
  • Processing Variabler: key, mousePressed, mouseX, mouseY, frameCount.
  • Scope og kode-blok: kode-blokke og scope er indenfor krølle-paranteserne { }. Scope betydning for hvor og hvornår variabeler eksisterer
  • Variabler: deklaration, værditildeling
  • Primitive datatyper: int, float, boolean, char ... (der er også. byte, short, long og double)
  • Primitive datatype Konvertering: …
  • Ikke-primitive datatyper: F.eks. String's, arrays og klasser
  • String: brug af +, length, split
  • Arrays: oprettelse, ændring, indeksering, flere dimensioner og length
  • ArrayList: oprettelse, clear, add, get, remove og size

Betingelser: Sammenlignings operatorer f.eks. && || <> eller == og Sammenligninger

  • if-statements / if-else / if - else if - else
  • Løkker: for-loop og while-loops.
  • break: kommando til at hoppe ud af et loop
  • continue: kommando til at hoppe videre til næste “step” i loopet

Design af funktioner: returtype, input parametre og funktionsnavnet

Vigtig terminologi:

  • Klasser og Objekter:vigtige keywords er class og new
  • Indkapsling: det anses som god kodepraksis at indkapsle data i objekter og kun tilgå data igennem metoder.
  • Ansvarsområder: det er anses ligeledes som god kodepraksis at en klasse har et ansvarsområde.
  • Typer og typekonvertering: Med klasser kan man definere n
  • Polymorfi: Type-polymorfi er f.eks at man kan skrive A a1 = new B() og a1 = new C(); (der findes også override og overload af metoder)
  • Nedarving: keyword extends. Når en klasse har en "er en" relation
  • Komposition: Når en klasser har en "har en" relation
  • Konstruktører: Den "metode" som kaldes når et objekt skal skabes. Har altid samme navn som klassen og ingen returtype

Objektorienteret analyse og design (forkortet oop): En systematisk måde at udvikle oop på (se arbejdsprocessen og arbejdsgange ovenfor)

  • Udskrivning/Læsning fra fil: loadTable/saveTable og loadStrings/saveStrings
